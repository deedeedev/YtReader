# Fix Plan: SubtitleEntity Normalization

## Problem

The current `SubtitleEntity` mixes 5 distinct concerns in a single entity with 25+ fields:

1. **Content & metadata** (9 fields): `videoId`, `videoUrl`, `title`, `channelName`, `languageCode`, `subtitleTrackId`, `isAutoGenerated`, `trackIdentity`, `content`, `studyContent`
2. **Reading state** (7 fields): `lastTimestamp`, `lastOpenedAt`, `readingProgressPercent`, `isRead`, `currentPage`, `totalPages`, `lastStudyScroll`
3. **Display preferences** (2 fields): `fontSize`, `fontFamily`
4. **AI cleaning state** (6 fields): `aiCleaningInProgress`, `aiCleaningSourceText`, `aiCleaningPendingResult`, `aiCleaningErrorSummary`, `aiCleaningErrorLog`, `aiCleaningUpdatedAt`
5. **Other** (4 fields): `id`, `isInLibrary`, `createdAt`, `uploadDate`, `highlights`

---

## Proposed Solution: Normalize into 3 Entities

### Entity 1: `SubtitleEntity` (Core — stays in `subtitles` table)

Keeps identity, content, display preferences, and library state. Display preferences (`fontSize`, `fontFamily`) and `highlights` stay here because they're always needed alongside content and are simple scalar values.

**Fields (16):**
- `id` (PK, autoGenerate)
- `videoId` TEXT NOT NULL
- `videoUrl` TEXT NOT NULL
- `title` TEXT NOT NULL
- `channelName` TEXT NOT NULL
- `languageCode` TEXT NOT NULL
- `subtitleTrackId` TEXT (nullable)
- `isAutoGenerated` INTEGER NOT NULL
- `trackIdentity` TEXT NOT NULL
- `content` TEXT NOT NULL
- `studyContent` TEXT (nullable)
- `isInLibrary` INTEGER NOT NULL
- `fontSize` REAL NOT NULL
- `createdAt` INTEGER NOT NULL
- `uploadDate` INTEGER NOT NULL
- `fontFamily` TEXT NOT NULL
- `highlights` TEXT NOT NULL

**Indices (unchanged):**
- `index_subtitles_videoId_trackIdentity` UNIQUE on `(videoId, trackIdentity)`
- `index_subtitles_createdAt` on `(createdAt)`
- `index_subtitles_isInLibrary` on `(isInLibrary)`
- `index_subtitles_channelName` on `(channelName)`

**Removed index:** `index_subtitles_lastOpenedAt` — moved to `subtitle_reading_states`

### Entity 2: `SubtitleReadingStateEntity` (new table: `subtitle_reading_states`)

One-to-one with `subtitles`. FK cascades on delete.

**Fields (8):**
- `subtitleId` INTEGER NOT NULL PRIMARY KEY, FK → `subtitles(id)` ON DELETE CASCADE
- `lastTimestamp` INTEGER NOT NULL DEFAULT 0

- `lastStudyScroll` INTEGER NOT NULL DEFAULT 0

**Indices:**
- `index_subtitle_reading_states_lastOpenedAt` on `(lastOpenedAt)`

### Entity 3: `AiCleaningStateEntity` (new table: `ai_cleaning_states`)

One-to-one with `subtitles`. FK cascades on delete.

**Fields (7):**
- `subtitleId` INTEGER NOT NULL PRIMARY KEY, FK → `subtitles(id)` ON DELETE CASCADE

- `aiCleaningSourceText` TEXT (nullable)
- `aiCleaningPendingResult` TEXT (nullable)

## Migration Strategy (23 → 24)

**Critical:** SQLite < 3.35.0 (Android API < 31) does NOT support `ALTER TABLE DROP COLUMN`. Since `minSdk = 26`, we must use the **table-recreation pattern** for `subtitles`.

The `observeVideoRows` query in `SubtitleDao` is the trickiest part — it uses subqueries that reference `lastOpenedAt`, `readingProgressPercent`, `currentPage`, `totalPages` from the `subtitles` table. After migration, these must be JOINed from `subtitle_reading_states`.

### Migration SQL

```sql
-- Step 1: Create new tables
CREATE TABLE IF NOT EXISTS `subtitle_reading_states` (
    `subtitleId` INTEGER NOT NULL PRIMARY KEY,
    `lastTimestamp` INTEGER NOT NULL DEFAULT 0,
    `lastOpenedAt` INTEGER NOT NULL DEFAULT 0,
    `readingProgressPercent` INTEGER NOT NULL DEFAULT 0,
    `isRead` INTEGER NOT NULL DEFAULT 0,
    `currentPage` INTEGER NOT NULL DEFAULT 0,
    `totalPages` INTEGER NOT NULL DEFAULT 0,
    `lastStudyScroll` INTEGER NOT NULL DEFAULT 0,
    FOREIGN KEY(`subtitleId`) REFERENCES `subtitles`(`id`) ON UPDATE NO ACTION ON DELETE CASCADE
);

CREATE TABLE IF NOT EXISTS `ai_cleaning_states` (
    `subtitleId` INTEGER NOT NULL PRIMARY KEY,
    `aiCleaningInProgress` INTEGER NOT NULL DEFAULT 0,
    `aiCleaningSourceText` TEXT,
    `aiCleaningPendingResult` TEXT,
    `aiCleaningErrorSummary` TEXT,
    `aiCleaningErrorLog` TEXT,
    `aiCleaningUpdatedAt` INTEGER NOT NULL DEFAULT 0,
    FOREIGN KEY(`subtitleId`) REFERENCES `subtitles`(`id`) ON UPDATE NO ACTION ON DELETE CASCADE
);

-- Step 2: Copy data into new tables
INSERT INTO `subtitle_reading_states` (
    subtitleId, lastTimestamp, lastOpenedAt,
    readingProgressPercent, isRead, currentPage, totalPages, lastStudyScroll
)
SELECT id, lastTimestamp, lastOpenedAt,
    readingProgressPercent, isRead, currentPage, totalPages, lastStudyScroll
FROM subtitles;

INSERT INTO `ai_cleaning_states` (
    subtitleId, aiCleaningInProgress, aiCleaningSourceText,
    aiCleaningPendingResult, aiCleaningErrorSummary,
    aiCleaningErrorLog, aiCleaningUpdatedAt
)
SELECT id, aiCleaningInProgress, aiCleaningSourceText,
    aiCleaningPendingResult, aiCleaningErrorSummary,
    aiCleaningErrorLog, aiCleaningUpdatedAt
FROM subtitles;

-- Step 3: Create indices on new tables
CREATE INDEX IF NOT EXISTS `index_subtitle_reading_states_lastOpenedAt`
    ON `subtitle_reading_states` (`lastOpenedAt`);

-- Step 4: Recreate subtitles table without removed columns
CREATE TABLE IF NOT EXISTS `subtitles_new` (
    `id` INTEGER PRIMARY KEY AUTOINCREMENT NOT NULL,
    `videoId` TEXT NOT NULL,
    `videoUrl` TEXT NOT NULL,
    `title` TEXT NOT NULL,
    `channelName` TEXT NOT NULL,
    `languageCode` TEXT NOT NULL,
    `subtitleTrackId` TEXT,
    `isAutoGenerated` INTEGER NOT NULL,
    `trackIdentity` TEXT NOT NULL,
    `content` TEXT NOT NULL,
    `studyContent` TEXT,
    `isInLibrary` INTEGER NOT NULL,
    `fontSize` REAL NOT NULL,
    `createdAt` INTEGER NOT NULL,
    `uploadDate` INTEGER NOT NULL,
    `fontFamily` TEXT NOT NULL,
    `highlights` TEXT NOT NULL
);

INSERT INTO `subtitles_new` (
    id, videoId, videoUrl, title, channelName, languageCode,
    subtitleTrackId, isAutoGenerated, trackIdentity, content,
    studyContent, isInLibrary, fontSize, createdAt, uploadDate,
    fontFamily, highlights
)
SELECT
    id, videoId, videoUrl, title, channelName, languageCode,
    subtitleTrackId, isAutoGenerated, trackIdentity, content,
    studyContent, isInLibrary, fontSize, createdAt, uploadDate,
    fontFamily, highlights
FROM subtitles;

-- Step 5: Swap tables
DROP TABLE subtitles;
ALTER TABLE subtitles_new RENAME TO subtitles;

-- Step 6: Recreate indices on subtitles
CREATE UNIQUE INDEX IF NOT EXISTS `index_subtitles_videoId_trackIdentity`
    ON `subtitles` (`videoId`, `trackIdentity`);
CREATE INDEX IF NOT EXISTS `index_subtitles_createdAt`
    ON `subtitles` (`createdAt`);
CREATE INDEX IF NOT EXISTS `index_subtitles_isInLibrary`
    ON `subtitles` (`isInLibrary`);
CREATE INDEX IF NOT EXISTS `index_subtitles_channelName`
    ON `subtitles` (`channelName`);
-- Removed: index_subtitles_lastOpenedAt (now on subtitle_reading_states)
```

**No data is lost.** All 13 removed columns are preserved in the two new tables before the old columns are dropped. The only "loss" is the `index_subtitles_lastOpenedAt` index on the `subtitles` table itself, which is replaced by an index on `subtitle_reading_states`.

### Phase 2: Update SubtitleEntity & SubtitleDao

**Step 2.1:** Update `SubtitleEntity.kt` — remove the 13 fields moved to new entities.

**Step 2.2:** Update `SubtitleDao.kt`:

**Remove these methods** (moved to new DAOs):
- `updateLastTimestamp`, `updateLastOpenedAt`, `updateReadingProgress`, `updateLastStudyScroll`
- `resetReadingProgressForVideo`, `markVideoAsRead`, `getMostRecentlyOpened`
- `markAiCleaningQueued`, `storeAiCleaningResult`, `storeAiCleaningFailure`, `cancelAiCleaning`, `clearAiCleaningResult`, `clearAiCleaningError`

**Update `observeVideoRows` query:**
The query currently reads `lastOpenedAt`, `readingProgressPercent`, `currentPage`, `totalPages` from `subtitles` via subqueries. These must now JOIN with `subtitle_reading_states`. Example change:

```sql
-- Before: reads from subtitles s
SELECT s.readingProgressPercent FROM subtitles s WHERE s.videoId = ...

-- After: joins with reading state
SELECT rs.readingProgressPercent
FROM subtitles s
LEFT JOIN subtitle_reading_states rs ON rs.subtitleId = s.id
WHERE s.videoId = ...
```

This affects the `agg` subquery (which uses `MAX(lastOpenedAt)`) and multiple scalar subqueries in `observeVideoRows`.

**Update `upsertByIdentity`:** Remove `fontSize`, `fontFamily` from the update if they stay in `SubtitleEntity` (they do, so no change needed here).

**Update `replaceContentForRedownload`:** Currently resets reading progress fields. After normalization, this must:
- Update content fields on `subtitles`
- Reset reading state in `subtitle_reading_states` (or use a `@Transaction` method)

**Update `observeAllAccessibleSubtitles`:** This currently returns `Flow<List<SubtitleEntity>>`. The `lastOpenedAt` ORDER BY must change to a JOIN.

**Keep `updateLibraryVisibility` and `deleteByVideoId`** as-is (they only reference `isInLibrary` and `videoId`).

**Step 4.1:** Update `SubtitleRepository.kt`:

Create a combined data class for consumers that need the full picture:

```kotlin
data class SubtitleWithStates(
    val subtitle: SubtitleEntity,
    val readingState: SubtitleReadingStateEntity?,
```

**Update existing methods:**
- Methods that return `SubtitleEntity` where reading state is also needed → return `SubtitleWithStates` or add parallel calls
- `updateLastTimestamp`, `updateLastOpenedAt`, etc. → delegate to `SubtitleReadingStateDao`
- AI cleaning methods → delegate to `AiCleaningStateDao`

**Key decision point:** For `observeById` which is used by `ReaderViewModel`, we need both reading state and AI cleaning state. Options:
- (A) Use Room `@Relation` for an automatic join (requires `@Embedded` + `@Relation` annotation)
- (B) Manual combination in repository using `combine()` on flows
- **Recommendation: Option B** — simpler, no Room annotation magic, and both flows are already observed

```kotlin
fun observeSubtitleWithStates(id: Long): Flow<SubtitleWithStates?> {
    return combine(
        subtitleDao.observeById(id),
        readingStateDao.observeBySubtitleId(id),
        aiCleaningStateDao.observeBySubtitleId(id)
    ) { sub, reading, cleaning ->
        sub?.let { SubtitleWithStates(it, reading, cleaning) }
    }
}
```

**Step 5.5:** `observeVideoRows` in `SubtitleDao` — the big query

The `agg` subquery currently does:
```sql
SELECT videoId, MAX(createdAt) AS lastDownloaded, MAX(lastOpenedAt) AS lastOpenedAt, MAX(isInLibrary) AS isInLibrary
FROM subtitles WHERE ... GROUP BY videoId
```

After normalization, `lastOpenedAt` lives in `subtitle_reading_states`:
```sql
SELECT s.videoId, MAX(s.createdAt) AS lastDownloaded,
       COALESCE(MAX(rs.lastOpenedAt), 0) AS lastOpenedAt,
       MAX(s.isInLibrary) AS isInLibrary
FROM subtitles s
LEFT JOIN subtitle_reading_states rs ON rs.subtitleId = s.id
WHERE ...
GROUP BY s.videoId
```

Similarly, all scalar subqueries reading `readingProgressPercent`, `currentPage`, `totalPages` need JOINs.

## Risks & Considerations

1. **Migration is the highest-risk step.** The table-recreation pattern for `subtitles` is well-established but must be tested carefully. The schema JSON (v23) and Room's schema validation will catch mismatches.

2. **The `observeVideoRows` query is complex** (~120 lines of SQL with subqueries). Rewriting it with JOINs to `subtitle_reading_states` requires careful testing against the existing test suite.

3. **Performance:** The JOIN queries add overhead. However, `subtitle_reading_states` has a primary key of `subtitleId` which makes the JOIN an index seek — negligible cost for the typical data volumes (< 10k subtitles).

4. **Reading state row creation:** When a new subtitle is inserted, no reading state row exists yet. All consumers must handle `null` reading state gracefully (the entity has sensible defaults). Consider inserting a default reading state row on subtitle creation, or using `COALESCE` in queries.

5. **Backward compatibility for UI:** `LibraryVideoRow` already serves as a DTO that decouples the query from the entity. The library UI won't need changes — only the SQL query behind it.
